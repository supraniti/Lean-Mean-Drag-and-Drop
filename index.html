Lean & Mean Drag and Drop is a small script for dragging, dropping, sorting and reordering html structures

Features
Supports nested structures ('nestable sortables')
Smooth transitions
Auto scroll while dragging
Lightweight (~3.5kb gzipped)
No dependencies
Super easy to integrate with your own app
Supports touch events
Compatible with all modern browsers
Can work with any type of layout
Installation
                
//Load LMDD css and js files
<link href="../css/lmdd.min.css" rel="stylesheet">
<script src="../js/lmdd.min.js"></script>

//Initialize LMDD instance with your preferred options
<script>lmdd.set(document.getElementById('markupID'),{optionsObject});</script>

  • speaker_notes

    As seen in example code, the basic script is essentially the same for all cases, set LMDD on an element (containing all that has to be dragged or nested in), and declare the relevant classes (containerClass,draggableItemClass & handleClass). The rest is done through proper HTML markup.

Options

//LMDD Options default values:
lmdd.set(document.getElementById('someID'), {
    containerClass:'lmdd-container',
    draggableItemClass: 'lmdd-draggable',
    handleClass: false,
    dragstartTimeout: 50,
    calcInterval: 200,
    revert: true,
    nativeScroll: false,
    mirrorMinHeight: 100,
    mirrorMaxWidth: 500,
    positionDelay: false,
    dataMode: false
});
Option Description
containerClass Containers act as drop zones for draggable items
draggableItemClass Draggable items can be moved inside and between containers
handleClass Restricts drag start to a specific element which acts as a drag "handle"
dragstartTimeout Delays the drag operation for a short time to distinct it from other user intentions (such as selecting text)
calcInterval Time interval in which LMDD evaluates the dragged element position, Short interval means more responsive experience (and more CPU...)
revert When set to true the draggable will revert to its original position when the cursor is out of the container bounds
nativeScroll LMDD uses its own auto-scroll function, you can eliminate it and use the native browser scroll
mirrorMinHeight, mirrorMaxWidth Scale down the mirror size when it exceeds the maximum width set
positionDelay When set to true - position will not be recalculated when the mouse stops moving (Prevents flickering on deep nested structures)
dataMode When set to 'true' LMDD will undo all DOM mutations when the drag event ends, This is useful for integrating LMDD with Angule/React/Vue etc.
Additional Classes

'lmdd-block' - To animate transitions LMDD makes a copy of all movable elements on its scope and set them on relative position,
this is an expensive process and might (in some cases) alter the way your markup renders while dragging.
To prevent this from happening, and save CPU on drag operations, add the class 'lmdd-block' to the parts of your markup which are not needed
to be animated. This will stop LMDD from traversing down the DOM tree and keep all children of the element intact.
For example: 


<div id="basic-example"> (scope)
    <div class="grid"> (container)
        <div class="item lmdd-block">(draggable)
        <--! A lot of additional markup -->
        </div>
    </div>
</div>

'lmdd-clonner' & 'lmdd-dispatcher' - In order to clone an element while dragging you have to use two additional classes:
'lmdd-dispatcher' - for the container from which you want to clone (this container will not accept new elements)
'lmdd-clonner' - for the draggable item you want to clone.
For example: 


<div id="clone-example"> (scope)
    <div class="grid lmdd-dispatcher"> (dragging items out from this container will clone them)
        <div class="item lmdd-clonner">(this draggable item will be cloned when dragged)
        </div>
    </div>
    <div class="grid"> (container)
    </div>
</div>

Events

LMDD dispatches three events:
'lmddbeforestart': Dispatched before any changes has been made to the DOM. Does not contain a detail object.
'lmddstart': Dispatched when the drag operation starts. Contains a 'detail' object.
'lmddend':Dispatched when the drag operation starts. Contains a 'detail' object.
The detail object carries the following information:
dragType: "move" or "clone"
draggedElement: HTML element being dragged
from.container: HTML element that originally contained the dragged element
from.index: The original index of the dragged element
to.container: HTML element that currently contains the dragged element
to.index: The current index of the dragged element

Integrations
LMDD is fairly easy to integrate with any other script. I would recommend treating the drag operation as any other user input,
just some kind of a sophisticated mouse gesture.
Having this point of view in mind, the integration process should be as follows:
1. Let your app do whatever your app does...
2. Upon rendering your markup, set an LMDD instance on the relevant part of the DOM
3. Set the dataMode to 'true'. (Which will undo any changes LMDD made on the DOM when the drag operation ends)
4. Add an eventListener for the 'lmddend' event
5. Use an event handler to process the event details and implement the necessary changes on your app data/state

          Example #1 - Simple Sortable Lists

          Note LMDD is markup oriented, meaning your HTML markup determines the way drag and drop behaviour:

          Elements marked with class="your-draggable-class" will be draggable

          Elements marked with class="your-container-class" will act as containers and you will be able to drop draggables inside them

          HTML markup:

          <div id="simple-sortable-1">
  <div class="container">
      <div class="orange rectangle"></div>
      <div class="light-green rectangle"></div>
      <div class="pink rectangle"></div>
      <div class="deep-purple rectangle"></div>
      <div class="light-blue rectangle"></div>
      <div class="lime rectangle"></div>
  </div>
</div>

          Javascript:

          lmdd.init(document.getElementById('yourMarkupID'),{yourOptionsObject});

          Example #2 - Containers, sortable, and draggable items

          LMDD is markup oriented, meaning you use the markup to tell the app what to act like...

          • html element marked with 'lmdd-container' class will act as a drop target
          • html element marked with 'lmdd-draggable' class will set the element as draggable so you can move it around and drop it in containers
          • html element marked with 'lmdd-sortable' class means this is a sortable item (you can drop the draggable before or after this element
